# Governance

This document outlines the governance model for projects within the `colinstiles` GitHub. My goal is to ensure projects are maintained in a transparent and consistent manner.

This guide works in conjunction with our Contributing Guidelines and Code of Conduct.

## Roles and Responsibilities

*   **Contributors:** Anyone who opens an issue or pull request. Contributors are encouraged to participate in discussions and follow the guidelines in `CONTRIBUTING.md`.
*   **Maintainers:** Contributors who have earned trust and are given write access to one or more repositories. They are responsible for:
    *   Guiding the project's technical and strategic direction.
    *   Reviewing and merging pull requests.
    *   Triaging issues and providing support to users.
    *   Enforcing the Code of Conduct.
    *   Protecting sensitive data and credentials.
    *   Keeping automation, documentation, and dependencies maintainable.

## Project Decisions

Decisions are made through consensus among maintainers, typically through discussion in relevant GitHub issues and pull requests. All decisions should prioritize:

*   **Public Service Value:** Does this serve the agency's mission and the public good?
*   **Data Quality & Reproducibility:** Is the work accurate and easy to replicate?
*   **Security & Privacy:** Does it protect sensitive information?
*   **Clarity:** Is the code and documentation clear and easy to understand?
*   **Sustainability:** Is the solution maintainable over the long term?

## Repository Ownership

Each repository may define its own owners, reviewers, and release practices, typically in its `README.md` file. When a repository does not define its own process, these default expectations apply, and organization administrators act as the primary maintainers.

## Amending This Document

Changes to this governance document are welcome. Please open a pull request in this repository to propose an amendment. The change will be reviewed and approved by organization administrators.
